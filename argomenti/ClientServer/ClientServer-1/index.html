Dai Mainframe al Client/Server

Quando, Descrizione

Mainframe I mainframe sono potenti calcolatori centralizzati che dagli anni ’60 in poi venivano usati in grandi aziende per risolvere diverse operazioni di calcolo. Occupavano intere stanze ed erano assemblati sotto forma di armadi metallici (“frame” = “armatura”). Fino all’arrivo dei PC negli anni ’80-’90 i mainframe sono stati il simbolo dell’elaborazione di fascia alta.
L’utilizzo storico dei mainframe è la gestione dei carichi transazionali, ma di recente capacità di gestire grandi quantità di operazioni di input/output ha portato i mainframe anche nelle applicazioni de gestione di Big Data.
Composti da numerosi processori (oltre un centinaio di CPU), numerosi banchi di memorie RAM (centinaia di gigabyte).
Molti mainframe contenevano dei processori che non erano accessibili all’utente, ma che si attivavano solo nel caso in cui delle CPU si bloccassero. Client/Server (C/S) Il modello C/S è costituito da un insieme di processi in esecuzione su diversi host:
Server: processi che gestiscono una o più risorse (host).
Client: processi che richiedono l’accesso ad alcune risorse.
Un processo server può diventare a sua volta diventare client ed essere contemporaneamente sia client che server. Differenza server e servizio:
-Servizio:entità astratta da uno o più server;
-Server: insieme di macchine, eterogenee, che ospitano dei servizi;
Esempi di servizi tipici delle architetture client/server:
- HTTP (Porta 80) e HTTPS (porta 443): utilizzati per la trasmissione d’informazioni sul web.
- FTP (Porte 21/20): utilizzati per il trasferimento di dati tra diversi host.
- Telnet (Porta 23): utilizzato per il controllo di un computer da remoto.

Schema funzionamento del modello Client/Server:
1. Il Client invia una richiesta al Server (richiesta di servizio).
2. Il server (in stato di listening) riceve la richiesta.
3. Server esegue il servizio richiesto (mediante un thread concorrente).
4. Server invia una risposta ed eventuali dati al client (richiedente).
5. Il client riceve la risposta ed eventuali dati.
