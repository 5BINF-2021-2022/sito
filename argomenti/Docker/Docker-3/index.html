Come funziona Docker?

"Docker risolve un problema fondamentale ovvero la necessità di avere degli ambienti di sviluppo identici per ogni singolo componente applicativo"

Da cosa è composto?

Docker per funzionare ha bisogno di due strumenti:

  • il docker Engine
  • dei container


Il Docker Engine

Se nelle Macchine Virtuali abbiamo bisogno di un hypervisor per eseguire le nostre macchine, in Docker abbiamo bisogno del Docker Engine che ci permette di eseguire i CONTAINER.
Quando parliamo di docker in realtà ci riferiamo al Docker Engine, un motore di containerizzazione open source , usato per creare e conteinerizzare le proprie applicazioni.
Il Docker Engine può essere scaricato dal sito ufficiale inizialmente creata per linux ora disponibile anche per windows e mac.
Questa tecnologia agisce come un'applicazione client/server infatti possiede:

  • Un server con un processo demone di lunga durata chiamato dockerd
  • Delle API che specificano le interfacce che i programmi possono utilizzare per comunicare e istruire il demone
  • Un interfaccia comandi (CLI) utilizzabile mediante il comando docker


I Container

Un conteiner è un'unità standard di software che impacchetta il codice e tutte le sue dipendenze in modo che l'applicazione venga eseguita in modo rapido e affidabile da un ambiente di elaborazione all'altro.
Un'immagine di un conteiner è un pacchetto software leggero, autonomo ed eseguibile che include tutto il necessario per eseguire un'applicazione:

  • codice,
  • runtime,
  • strumenti di sistema,
  • librerie di sistema,
  • impostazioni,

Le immagini diventano conteiner in fase di esecuzione sul Docker Engine. Il software containerizzato funzionerà sempre allo stesso modo, indipendentemente dall'infrastruttura, I container isolano il software dal suo ambiente e assicurano che funzioni in modo uniforme nonostante le differenze, ad esempio, tra sviluppo e staging.
