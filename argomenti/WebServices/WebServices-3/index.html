Pila protocollare

Il fatto che l'industria si sta muovendo verso standard di programmazione (Java) e di scambio dei dati (XML) comuni, ha favorito l'ascesa dei Web Services.
La pila protocollare dei Web service e' l'insieme dei protocolli di rete utilizzati per definire, localizzare, realizzare e far interagire tra di loro i Web service.
Trasporto del servizio:
responsabile per il trasporto dei messaggi tra le applicazioni in rete, include il protocollo HTTP. XML:
Extensible Markup Language, o XML, fornisce le fondamenta per i Web Servicese e quella tecnologia dei Web services su cui si basano tutte le altre tecnologie. Simile a HTML, XML usa tag di programmazione per definire la proprieta' dei dati sulle pagine Web e altri documenti.
SOAP (Simple Object Access Protocol):
e' un protocollo che controlla lo scambio di messaggi tra componenti software.
Permette la comunicazione tra applicazioni attraverso Internet, indipendentemente dalla piattaforma su cui queste applicazioni girano.
I parametri d'ingresso e uscita del metodo del componente remoto sono definiti in un documento XML che viene incapsulato in un messaggio HTTP e inviato al Web Application Server del provider del servizio.
Un dispatch SOAP, decodifica il messaggio XML, invoca il metodo richiesto e prepara una risposta, codificata in XML, che invia al richiedente.
Descrizione del servizio (WSDL):
Per integrare automaticamente un Web Service a livello di programmazione e' necessario un documento che descriva tutti i dettagli tecnici relativi al servizio come ad esempio, l'interfaccia (nomi dei metodi, i tipi degli argomenti e dei valori di ritorno) e l'utilizzo.
Il linguaggio usato per scrivere questo documento e' chiamato WSDL (Web Services Description Language, basato su XML.
Elenco dei servizi (UDDI):
Universal Discovery, Description and Integration (UDDI), mette a disposizione directory services pubblicamente raggiungibili per Web services basati su XML.
Quando richiesto dalle applicazioni, i registri UDDI forniscono informazioni su servizi nuovi, sullo stato di servizi particolari e sulla disponibilita' dei Web service compatibili.
WS-Security:
il protocollo standard Web Services Security protocol permette l'autenticazione degli utenti e la confidenzialita' dei messaggi scambiati con l'interfaccia del Web service.
