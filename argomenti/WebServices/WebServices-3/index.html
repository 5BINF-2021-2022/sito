Pila protocollare

Il fatto che l'industria si sta muovendo verso standard di programmazione (Java) e di scambio dei dati (XML) comuni, ha favorito l'ascesa dei Web Services.
La pila protocollare dei Web service e' l'insieme dei protocolli di rete utilizzati per definire, localizzare, realizzare e far interagire tra di loro i Web service.
Trasporto del servizio:
Responsabile per il trasporto dei messaggi tra le applicazioni in rete, include il protocollo HTTP.
XML:
Extensible Markup Language, fornisce le fondamenta per i Web Servicese e quella tecnologia dei Web services su cui si basano tutte le altre tecnologie. XML è un linguaggio di markup simile al linguaggio HTML: si tratta di un insieme di regole per la codifica dei documenti tramite tag, in un formato leggibile sia per gli utenti che per le macchine.
La differenza tra XML e HTML è legata ai tag: infatti mentre HTML si basa su numero limitato di tag, XML consente di utilizzare un numero illimitato di tag, definibili secondo le specifiche esigenze del progetto.
SOAP (Simple Object Access Protocol):
E' un protocollo che controlla lo scambio di messaggi tra componenti software.
Permette la comunicazione tra applicazioni attraverso Internet, indipendentemente dalla piattaforma su cui queste applicazioni girano.
I parametri d'ingresso e uscita sono definiti in un documento XML che viene incapsulato in un messaggio HTTP e inviato al Web Application Server del provider del servizio.
Un dispatch SOAP, decodifica il messaggio XML, invoca il metodo richiesto e prepara una risposta che invia al richiedente.
Descrizione del servizio (WSDL):
Il Web Services Description Language (WSDL) è un linguaggio formale in formato XML utilizzato per la creazione di "documenti" che descrivono i Web service.
Mediante WSDL può essere infatti descritta l'interfaccia pubblica di un Web service che indica come interagire con un determinato servizio.
Un "documento" WSDL contiene infatti, relativamente al Web service descritto, informazioni su:

  • cosa può essere utilizzato (le "operazioni" messe a disposizione dal servizio);
  • viene specificato il protocollo di comunicazione da utilizzare per accedere al servizio, il formato dei messaggi accettati in input e restituiti in output dal servizio;
  • dove si trova il servizio nella rete.

Elenco dei servizi (UDDI):
Universal Discovery, Description and Integration (UDDI), mette a disposizione directory services pubblicamente raggiungibili per Web services basati su XML.
Quando richiesto dalle applicazioni, i registri UDDI forniscono informazioni su servizi nuovi, sullo stato di servizi particolari e sulla disponibilita' dei Web service compatibili.
WS-Security:
il protocollo standard Web Services Security protocol permette l'autenticazione degli utenti e la confidenzialita' dei messaggi scambiati con l'interfaccia del Web service.
